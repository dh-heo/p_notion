# 리팩터 전후 수동 검증 체크리스트

자동 테스트(`npm run test:unit`)는 브라우저 DOM에 의존하지 않는 순수 로직만 고정한다.
아래 항목들은 contentEditable/DOM/시각 동작이라 단위 테스트로 덮을 수 없으므로,
리팩터 **전후로 `npm run dev` 후 브라우저에서 동일하게 동작하는지** 눈으로 확인한다.

검증 방법: 각 항목을 리팩터 **전에 한 번** 수행해 동작을 기억하고, **후에 다시** 수행해 차이가 없는지 본다.

## A. 기본 편집 (RichText)
- [ ] 문단에 타이핑 → Enter로 분할 → 새 블록 생성
- [ ] 블록 시작에서 Backspace → 이전 블록과 병합
- [ ] `# `/`## `/`### ` → 헤딩, `- ` → 불릿, `1. ` → 번호, `[] ` → 할일, `----` → 구분선
- [ ] `->` / `<-` 자동 치환 → `→` / `←`
- [ ] 한글 IME로 Enter 시 줄이 **한 번만** 추가되는지(중복 없음)
- [ ] `/` 슬래시 메뉴 열림 → 항목 선택 → 블록 변환
- [ ] `[[` 멘션 메뉴 → 페이지 선택 → 앵커 삽입, 클릭 시 이동
- [ ] ⌘/Ctrl+B/I/U 서식, ⌘/Ctrl+K 링크(선택 있을 때)
- [ ] ↑/↓/←/→ 블록 간 캐럿 이동(비텍스트 블록은 건너뜀)

## B. 표 블록 (TableBlock — 항목1 분할의 핵심 대상)
- [ ] 표 렌더 정상 (흰 화면 없음). 빈/깨진 content여도 self-heal되어 기본 2×2
- [ ] 텍스트 셀 편집 → 다른 셀로 마우스 이동해도 입력 중 텍스트 안 지워짐
- [ ] Tab/화살표로 셀 이동(텍스트 셀은 가장자리에서만 이동)
- [ ] 셀에서 `[[` 멘션 동작
- [ ] 열 머리글 `⌄` 메뉴: 정렬 asc/desc, 우측 열 삽입, 열 숨기기, 삭제, 필터, 천 단위 쉼표, 범주/텍스트 변환
- [ ] 열 숨기기 후 gap 마커/하단 "숨긴 열 N" 팝오버로 다시 보이기
- [ ] 열 드래그로 순서 변경(데이터 같이 이동)
- [ ] 범주(select) 열: 칩 색상, 옵션 선택/생성
- [ ] 천 단위 쉼표: 숫자 셀 미포커스 시 1000단위 표시, 포커스 시 원본
- [ ] 행 gutter: 아래 행 삽입/삭제, "머리글" 승격
- [ ] 셀 영역 드래그 선택 → 복사 → Excel/Sheets에 표로 붙여짐
- [ ] CSV 내보내기(BOM, 숨긴 열 제외)
- [ ] 스프레드시트 영역 붙여넣기 → 표 확장/생성

## C. 기타 블록
- [ ] 코드 블록: 편집, 언어 변경, 잠금 시 read-only
- [ ] 이미지/파일 드롭 → 블록 생성
- [ ] YouTube/북마크/수식(KaTeX)/토글 정상 렌더·편집

## D. 시각/레이아웃 (components.css — 항목2 분할의 핵심 대상)
- [ ] 사이드바(트리, 최근 편집, 색상 플래그, 휴지통) 정상
- [ ] 에디터 폭/여백, 텍스트 블록 스타일(인용/콜아웃/토글/북마크/수식)
- [ ] 슬래시 메뉴·인라인 툴바·멘션 메뉴·검색 모달 위치와 스타일
- [ ] 좁은 화면(≤720px): 사이드바 오버레이/햄버거, 넓은 화면 접기 토글
- [ ] 로그인/설정 화면
- [ ] 다중 블록 선택 툴바(불릿/번호/굵게/색)

## E. 페이지/저장 (store — 항목5 분할 대상)
- [ ] 페이지 추가/이름변경/아이콘/색상/삭제·복원(휴지통)
- [ ] 편집 후 새로고침 시 내용 유지(디바운스 저장)
- [ ] 마지막 본 페이지 복원, 편집 잠금 토글
- [ ] 검색(⌘K), 백링크 표시
